MH-SOAR+ User Guide

What is MH-SOAR+?

MH-SOAR+ is the system used by stores to create orders, manage payments, and track what happens after checkout.

It replaces manual processes and helps store crews work faster, more accurately, and with fewer errors.

What MH-SOAR+ is used for

Store crews use MH-SOAR+ to:

  • Create customer orders

  • Add items and services to an order

  • Review order details before checkout

  • Process customer payments

  • Track order status after checkout

  • Print or view receipts when needed

Who uses MH-SOAR+?

MH-SOAR+ is designed for:

  • Store crews ‚Äì daily order creation and payment processing

  • Cashiers ‚Äì handling checkout and payments

  • Supervisors / Managers ‚Äì monitoring order progress and resolving issues

Each role may see slightly different screens, but the core flow is the same.

What MH-SOAR+ is not

To avoid confusion:

  • MH-SOAR+ is not an accounting system

  • MH-SOAR+ is not used to edit prices or service rules

  • MH-SOAR+ is not for customer self-use

It is a store operations tool focused on order processing.

Why MH-SOAR+ is important

Using MH-SOAR+ helps the store to:

  • Reduce manual errors

  • Speed up checkout

  • Ensure orders are properly recorded

  • Keep payment and order data consistent

  • Support smoother store testing and daily operations

Login and Access

To use MH-SOAR+, you must log in using your assigned credentials.

This ensures that:

  • orders are recorded under the correct store

  • activities are properly tracked

  • access is limited to authorized users only

How to log in to MH-SOAR+

Follow these steps to log in:

  1. Open the MH-SOAR+ app

  2. Enter your username and password

  3. Select the branch you are assigned to

  4. Tap Log In

Once logged in successfully, you will be taken to the main screen of the app.

Selecting a branch

After entering your credentials, you will be asked to select a branch.

  • The branch determines where orders will be recorded

  • Only branches you are authorized for will appear

  • Make sure you select the correct branch before logging in

‚ö†Ô∏è Important
Creating orders under the wrong branch may cause reporting and tracking issues.

Offline access

MH-SOAR+ supports offline access, but with an important requirement.

When offline access is allowed

You can use MH-SOAR+ offline if:

  • You have successfully logged in online at least once

  • The app has already verified your credentials

Once this is done, the app can be opened and used even without an internet connection.

When offline access is not allowed

Offline access will not work if:

  • You have never logged in online before

  • You logged out and have not reconnected online

  • The app was newly installed and not yet authenticated

‚ö†Ô∏è Important
First-time login must be done while connected to the internet.

Best practices for login

  • Always confirm you are logged in to the correct branch

  • Do not share your username or password

  • If login fails, check your internet connection first

  • Contact your supervisor or support if you cannot log in

What happens after login

Once logged in, you can:

  • Create new orders

  • Add items and services

  • Process payments

  • Continue working even if the connection becomes unstable (after initial login)

Sold Out/Z Reading

Sold Out / Z-Reading

The Sold Out / Z-Reading feature is used to record the daily sales summary of the store.

This includes:

  • Daily gross sales

  • Expenses for the day

  • Tender amounts

  • Uploading supporting images

  • Sending the record for syncing to iDempiere

This process is usually completed at the end of the day.

Step 1: Open Sold Out / Z-Reading

  1. From the menu, select Sold Out / Z-Reading

  2. A prompt will appear:

‚ÄúDo you want to create a new sold out record?‚Äù

  1. Select Yes

Step 2: Review the header details

You will be redirected to a page showing:

  • Document Number (auto-generated)

  • Branch Name

  • Sales Date

Editing the sales date

  • Click Edit if you need to update the date

  • Make sure the sales date matches the actual business date

Step 3: Add Sales Line

  1. Select Sales Line

  2. A prompt will appear:

‚ÄúWould you like to add an invoice?‚Äù

  • Select Yes ‚Üí to add expenses

  • Select No ‚Üí to proceed directly to Gross Sales

Step 4: Add invoices (Expenses)

If Yes was selected:

  • Encode all expenses for the day

  • Review the details carefully

  • Proceed once all expenses are recorded

If there are no expenses, you may skip this step.

Step 5: Enter Gross Sales

  • Encode the gross sales for the day

  • Ensure all amounts are accurate

Step 6: Save the Sold Out record

  1. Click Save

  2. Review the record details

‚ö†Ô∏è Important
Saving does not automatically send the record to iDempiere.

Step 7: Open Bank Account menu

  1. From the menu, select Bank Account

  2. Choose the appropriate bank account where cash will be deposited

Step 8: Encode Tender Types

  • Select the applicable Tender Type (example: Cash)

  • Enter the corresponding amount

  • Add prices only if applicable

Repeat until all tender types are encoded.

Step 9: Save Bank Account details

  • Click Save after encoding all tender amounts

Step 10: View Sold Out Report

  1. Go back to Sold Out / Z-Reading

  2. Select Sold Out Report

  3. Review the summary of:

    • Sales

    • Expenses

    • Tender amounts

Step 11: Attach images (Optional)

  1. Select Attach Images

  2. Upload supporting images such as receipts or reports

Step 12: Final save and confirmation

  1. Click Save

  2. A confirmation prompt will appear

  3. Select Yes to proceed

You will be redirected to Sold Out Transactions.

Step 13: Send transaction

  1. In Sold Out Transactions, select the record to be sent

  2. Click Send Transaction

  3. Click Send

The record will be synced to iDempiere.

‚úÖ Important
Only sent transactions will be reflected in the system.

Best practices

  • Always double-check amounts before saving

  • Confirm the correct sales date

  • Encode Bank Account and Tender details properly

  • Attach images when required

  • Send the transaction only after reviewing the Sold Out Report

Requisition

The Requisition feature allows store crews to request products from the main system.

This is used when the store needs to:

  • Replenish stock

  • Request additional products

  • Submit item quantities for approval and processing in iDempiere

Step 1: Open Requisition

  1. From the menu, select Requisition

  2. A prompt will appear:

‚ÄúDo you want to create a new requisition?‚Äù

  1. Click Yes

Step 2: Review Requisition Details

You will be redirected to a new page showing:

  • Document Number (auto-generated)

  • Branch

  • Date Needed

Editing the date needed

  • Click Edit if you need to change the date

  • Make sure the date reflects when the products are required

Step 3: Add Requisition Line

  1. Click Requisition Line

  2. You will be directed to the requisition line page

  3. For each item you want to request:

    • Enter the quantity beside the product

Step 4: Save Requisition Line

  • Click Save after adding all required quantities

Step 5: Review Requisition Report

  1. Click Requisition Report

  2. Review the summary of:

    • Requested items

    • Quantities

    • Branch and date needed

Step 6: Save Requisition

  1. Click Save

  2. A confirmation prompt will appear

  3. Select Yes

You will be redirected to Requisition Transactions.

Step 7: Send Transaction

  1. In Requisition Transactions, select the requisition record

  2. Click Send Transaction

  3. Click Send

The requisition will now be sent and synced to iDempiere.

‚úÖ Only sent requisitions will be processed by the system.

Important Reminders

  • Double-check quantities before saving

  • Make sure the correct date needed is selected

  • Review the requisition report before sending

  • Always send the transaction to sync with iDempiere

Inventory

The Inventory menu is used to update the store‚Äôs existing inventory quantities.

This is done when:

  • Checking actual stock on hand

  • Updating quantities in the system

  • Aligning physical inventory with system records

Updating Inventory

Step 1: Open Inventory

  1. From the menu, select Inventory

  2. You will be directed to a new page showing:

    • Document Number (auto-generated)

    • Branch

    • Date

Editing the date

  • Click Edit if you need to change the date

  • Ensure the date reflects when the inventory was counted

Step 2: Update Inventory Line

  1. Click Inventory Line

  2. You will be redirected to the inventory line page

  3. For each product:

    • Update the quantity based on the actual stock on hand

Step 3: Save Inventory Line

  • Click Save after updating all product quantities

Review and Finalize

Step 4: Review Inventory Report

  1. Click Inventory Report

  2. Review the summary of:

    • Products

    • Updated quantities

    • Branch and date

Make sure all quantities are correct.

Step 5: Save Inventory

  1. Click Save

  2. A confirmation prompt will appear

  3. Select Yes

You will be redirected to Inventory Transactions.

Sending Inventory for Syncing

Step 6: Send Transaction

  1. In Inventory Transactions, select the inventory record

  2. Click Send Transaction

  3. Click Send

The inventory update will be synced to iDempiere.

‚úÖ Only sent inventory records will be reflected in the system.

Important Reminders

  • Always count items carefully before updating quantities

  • Make sure you are updating inventory for the correct branch

  • Review the inventory report before saving

  • Send the transaction to ensure the update is reflected in iDempiere

SOAR Templates (Supervisor Access)

SOAR Templates control which items appear‚Äîand in what order‚Äîwhen using:

  • Sales

  • Expenses

  • Requisition

  • Inventory

This feature allows supervisors or authorized employees to customize templates used during daily operations.

‚ö†Ô∏è Important
This menu is restricted. Only supervisors or authorized personnel with the correct password can access and edit templates.

Accessing SOAR Templates

Step 1: Open the menu

  1. Tap the three horizontal bars (‚ò∞) on the top-left corner of the app

  2. Select SOAR Template

Step 2: Enter authorization password

  1. Enter the required password

  2. Once validated, you will be granted access to the template menu

Selecting a Template Type

After logging in, you will see template options for:

  • Sales Template

  • Expense Template

  • Requisition Template

  • Inventory Template

Select the template you want to manage.

Editing a Template

Step 3: Open the template

  1. Tap the template type (example: Sales Template)

  2. You will be directed to the template editor page

Step 4: Manage template items

Inside the template editor, authorized users can:

  • Sort items

    • Rearrange the order in which items appear

  • Add items

    • Select items to include in the template

üìå Note
Only items that already exist in the system product list can be added to a template.

Step 5: Review template changes

Before saving:

  • Check the item order

  • Confirm all required items are included

  • Make sure no incorrect items were added

Step 6: Save the template

  • Click Save to apply the changes

  • The updated template will now be used in:

    • Sales

    • Expenses

    • Requisition

    • Inventory (depending on the template edited)

Important Rules and Reminders

  • Only authorized users can edit templates

  • Templates affect what store crews see during transactions

  • Items must exist in the product list before they can be added

  • Changes apply immediately after saving

Always review templates carefully before saving.
